# Atomic Alphabet V1 Labeling Manual

Status: freeze candidate, not final decipherment.

This manual exists to keep future labeling reproducible. Statistical rules discovered by EVAComparisonLab are evidence to test, not instructions for painting new data.

```text

```

Retired symbols:

```text
a:2 j:2 g:2
```

Retired symbols must not reappear in the active export unless a documented V2 change explicitly reintroduces them.

## Labeling Order

1. Set physical rows.
2. Decide molecule boundaries and cut/join states.
3. Decide particle membership inside each molecule.
4. Label atoms by visual morphology only.
5. Order atoms inside each particle.
6. Order particles inside each molecule.
7. Save and only then run statistical audits.

## Anti-Circularity Rules

- Do not label an atom because a grammar rule expects it.
- Do not merge or split particles because a rule would become cleaner.
- Do not correct an exception unless the canvas shows an independent visual error.
- Do not use EVA spelling to choose atom identity.
- Do not use the current entropy report while labeling out-of-sample pages.

## Variant Policy

- Keep `h:1` and `h:2` separate in V1.
- Keep `c:1` and `c:2` separate but under watch.
- Keep `j:1` and `n:1` separate. Former `j:2` remains retired.
- Keep `g:1` and `m:1` separate in labels.
- Macro merges such as `MEDIAL_OP` are analytical only and must not modify DatasetCreator labels.

## Known Anomalies

Some rare orderings are kept as known anomalies after visual inspection. They should not reappear as pending "possible human errors", but they remain anomalies in the record rather than being renamed as normal.

Maintain them in:

```text
../research/audits/known-labeling-anomalies.tsv
```

Use stable `atom_ids`, not molecule IDs, because molecule numbering can change after recalculation.

## Visual Evidence

Generated by:

```powershell
npm.cmd run corpus:v2
```

Snapshot outputs are regenerated local artifacts and are not committed as canonical evidence:

- `artifacts/visual-snapshots/current/visual-snapshots.db`
- `artifacts/visual-snapshots/current/visual-snapshots.tsv`
- `artifacts/visual-snapshots/current/atoms/<symbol>/<image>/<atom-id>.svg`
- `artifacts/visual-snapshots/current/particles/<image>/<particle-id>.svg`
- `artifacts/visual-snapshots/current/molecules/<image>/<molecule-id>.svg`

Snapshots are evidence artifacts derived from the source DB, not new labels.

## Change Control

Any V2 change must document:

- symbol added, retired, split, or merged;
- visual reason;
- affected atom IDs or molecule IDs when known;
- whether the change happened before or after seeing a statistical rule;
- rerun date and report path.

## Out-Of-Sample Protocol

1. Freeze this manual.
2. Label the next page without reading new rule outputs.
3. Export separately.
4. Run existing V1 rules against the new page.
5. Report survival, failure, and not-observed counts.
6. Only then decide whether the alphabet needs V2.
