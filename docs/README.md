# Repository Documentation
These documents govern how VoynichLab is maintained. They explain the
repository and its safety boundaries; they are not scientific result bundles.

## Start here

| Document | Purpose |
|---|---|
| `MONOREPO-ARCHITECTURE.md` | Architectural zones, dependencies, and promotion flow |
| `REPO-INVENTORY.md` | Maintainer map and cleanup risk |
| `PUBLIC-VERIFIABILITY.md` | Inspectable, maintainer-replay, and clean-clone trust levels |
| `DATASETCREATOR-SAFETY.md` | No-damage rules for annotation data |
| `DATASETCREATOR-ARCHITECTURE.md` | Read-only map of the protected annotation product |
| `DEPLOYMENT.md` | Canonical portal deployment rules |
| `LOCAL-STATE-INVENTORY.md` | Ignored caches, builds, databases, and machine state |

Audit reports and cleanup plans describe the repository at a point in time.
Current operating rules take precedence when historical paths differ.
